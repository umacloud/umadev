# 值班交接检查清单 (On-Call Handover Checklist)

> 适用场景：值班轮换交接、跨团队值班交接、重大事件期间的临时交接、假期前交接。
> 通过标准：所有 MUST 项在交接会议中完成确认，交接记录双方签字认可。

---

## 1. 当前系统状态

### 1.1 告警与事故

- [ ] **活跃告警已同步** — 当前所有未恢复的告警已逐条说明，包含触发时间、影响范围、当前处理状态。
- [ ] **进行中的事故已交接** — 正在处理的 P0/P1 事故的时间线、当前进展、下一步行动已说明。
- [ ] **近期已恢复的告警已说明** — 过去 24 小时内恢复的告警已说明，标注是否有复发风险。
- [ ] **静默 / 抑制的告警已说明** — 当前被静默的告警及其原因、预期恢复时间已交代。
- [ ] **误报告警已标注** — 已知的误报告警已说明，避免接班人花时间排查。

### 1.2 系统健康度

- [ ] **核心服务状态** — 各核心服务的健康状态已确认（正常 / 降级 / 部分不可用）。
- [ ] **资源使用趋势** — CPU / 内存 / 磁盘 / 连接数的使用趋势已说明，标注是否有接近阈值的风险。
- [ ] **依赖服务状态** — 第三方服务和上下游服务的当前状态已确认。
- [ ] **SLI / SLO 消耗情况** — 当前 Error Budget 消耗百分比已说明，是否处于风险区间。

---

## 2. 已知风险与临时方案

- [ ] **已知风险清单** — 当前存在但尚未修复的风险项已列出，包含影响范围和优先级。
- [ ] **临时绕行方案** — 正在使用的临时修复 / 绕行方案已说明，标注方案的有效期和限制条件。
- [ ] **技术债务风险** — 可能在值班期间触发问题的技术债务已标注（如快满的磁盘、即将过期的证书）。
- [ ] **定时任务注意事项** — 值班期间将运行的重要定时任务（如结算 / 对账 / 备份）已说明。
- [ ] **即将过期的资源** — 域名 / 证书 / License / API Key 等在值班期间可能过期的资源已标注。

---

## 3. 变更与发布

- [ ] **近期变更已同步** — 过去 48 小时内的生产变更已说明，包含变更内容、影响范围、负责人。
- [ ] **计划中的变更** — 值班期间计划执行的变更已列出，包含时间窗口、执行人、回滚方案。
- [ ] **灰度发布状态** — 正在进行的灰度 / 金丝雀发布的当前比例和观察指标已说明。
- [ ] **变更冻结期** — 是否处于变更冻结期（如大促前）已确认，冻结期规则已说明。
- [ ] **Feature Flag 状态** — 正在灰度中的 Feature Flag 状态已说明，包含启停条件。

---

## 4. 运维工具与入口

### 4.1 监控面板

- [ ] **核心监控 Dashboard 入口** — Grafana / CloudWatch / Datadog 等核心面板的 URL 已共享。
- [ ] **业务监控面板入口** — 业务关键指标（GMV / 订单量 / 转化率等）面板 URL 已共享。
- [ ] **基础设施面板入口** — 集群资源 / 数据库 / 缓存 / 队列的监控面板 URL 已共享。

### 4.2 操作工具

- [ ] **告警管理平台入口** — PagerDuty / OpsGenie / 自建告警平台的登录方式和操作权限已确认。
- [ ] **日志查询平台入口** — ELK / Loki / CloudWatch Logs 的查询入口和常用查询语句已共享。
- [ ] **CI/CD 平台入口** — Jenkins / GitHub Actions / ArgoCD 等部署平台的访问方式已确认。
- [ ] **数据库管理工具** — 数据库连接方式（跳板机 / 管理平台）和只读 / 读写权限已确认。

### 4.3 文档入口

- [ ] **Runbook 入口已确认** — 常见故障处理手册的最新版本位置已确认。
- [ ] **架构文档入口** — 系统架构图、服务依赖图的位置已确认。
- [ ] **应急预案入口** — 重大故障应急预案的位置已确认。

---

## 5. 升级路径与联系人

- [ ] **升级联系人可达** — 一级升级人（技术 Lead）、二级升级人（技术总监）、三级升级人（CTO）的联系方式已确认且可达。
- [ ] **DBA 支持联系方式** — DBA 值班人员的联系方式和响应时间承诺已确认。
- [ ] **安全团队联系方式** — 安全事件的上报联系人和沟通渠道已确认。
- [ ] **业务方联系人** — 需要业务决策时的对接人联系方式已确认。
- [ ] **第三方支持联系方式** — 云厂商 / CDN / 支付渠道等第三方的技术支持联系方式和 SLA 已确认。
- [ ] **升级触发条件已明确** — P0 事件 5 分钟升级、P1 事件 15 分钟升级等规则已复述确认。

---

## 6. 未完成事项

- [ ] **待修复项清单** — 上一班次遗留的待修复项已列出，包含优先级和预期完成时间。
- [ ] **待观察项清单** — 需要持续观察的指标或行为已列出，包含观察频率和判定标准。
- [ ] **待跟进的工单** — 已创建但未关闭的事故工单 / 问题工单的状态已同步。
- [ ] **待回复的沟通** — 来自业务方 / 用户 / 上下游的待回复沟通已说明。

---

## 7. 特殊情况说明

- [ ] **业务高峰期预警** — 值班期间是否有大促 / 活动 / 流量高峰已告知，峰值预估已说明。
- [ ] **计划内维护** — 值班期间的计划内维护（如数据库维护窗口 / 网络调整）已标注。
- [ ] **跨时区协调** — 需要跨时区沟通的事项已说明，对方团队的工作时间已标注。
- [ ] **合规审计事项** — 值班期间是否有审计访问或合规检查已告知。

---

## 8. 交接确认

- [ ] **交接时间已记录** — 交接发生的日期和时间已记录。
- [ ] **交接双方已确认** — 交班人和接班人姓名已记录，双方均确认交接内容完整。
- [ ] **值班日历已更新** — 值班排班表中的当前值班人已更新。
- [ ] **告警通知已切换** — PagerDuty / OpsGenie 等告警通知的接收人已切换到接班人。
- [ ] **IM 群组已更新** — 值班 IM 群组中的值班人标识已更新。
- [ ] **VPN / 跳板机已测试** — 接班人的远程访问权限已验证可用。
- [ ] **交接记录已存档** — 本次交接记录已保存到团队知识库或值班日志中。

## Agent Checklist

供 AI Agent 在协助值班交接时使用的自检项：

- [ ] 已收集当前所有活跃告警和进行中事故的状态。
- [ ] 已核对已知风险和临时绕行方案的完整性。
- [ ] 已确认所有升级联系人的可达性。
- [ ] 已验证监控面板和运维工具的访问链接有效。
- [ ] 已确认交接双方对所有待跟进事项达成一致理解。
- [ ] 已生成交接记录并存档至团队知识库。
- [ ] 已提醒接班人检查值班期间的特殊事项（高峰期 / 维护窗口）。
