## Agent 评测与基准体系

### 目标
- 让 Agent 能力可量化、可回归、可持续优化。

### 适用范围
- 适用于研发阶段评测、发布门禁评估和线上质量回归。

### 评测维度
- 正确性：任务是否完成且结果正确。
- 稳定性：同类任务重复执行波动范围。
- 时效性：平均耗时与超时率。
- 成本：单任务 token 成本与工具调用成本。
- 安全性：是否触发策略违规或越权行为。

### 基准集构建
- 核心场景题集：覆盖高频业务任务。
- 边界场景题集：异常输入、缺失上下文、冲突约束。
- 红队场景题集：提示注入、数据泄露诱导、越权调用。

### 门禁规则
- 关键场景正确率不得低于既定阈值。
- 安全违规项必须为零才可发布。
- 成本与时延回退超阈值必须阻断发布。

### 执行清单
- 每次模型、提示词、工具变更都触发回归评测。
- 基准集按业务优先级分层维护并定期扩充。
- 评测报告必须包含质量、安全、时延、成本四维趋势。

### 验收标准
- 关键路径题集通过率达到阈值且稳定性达标。
- 安全红队场景无阻断级失败项。

### 常见失败模式
- 只看演示样例，不做标准化回归评测。
- 只看正确率，不跟踪安全与成本退化。

### 回滚策略
- 评测结果异常时，暂停发布并回滚到上一个评测合格版本。
- 重新执行全量基准评测后再恢复放量。
