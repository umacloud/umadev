# 开发：Excellent（）

## 设计环节深度知识库

### 目标
- 建立可扩展、可复用、可验证的交互与视觉规范体系。

### 核心原则
- 信息优先：先定义信息架构，再定义视觉风格。
- 状态完整：加载、空态、错误、权限、成功反馈必须齐备。
- 任务导向：页面围绕任务完成率而不是元素堆叠。
- 一致性：相同语义使用相同交互模式与文案规则。

### 设计交付清单
- 页面级流程图与关键路径
- 组件状态矩阵
- 表单交互规则
- 错误信息标准模板
- 可访问性检查项（键盘、对比度、语义）

### 质量检查
- 每个页面至少有一个主任务与一个次任务。
- 表单必须具备即时校验与提交兜底校验。
- 高风险动作必须二次确认并提供撤销方案。

### 常见失败模式
- 只画正常态，不定义异常态。
- 视觉一致但交互不一致，导致学习成本升高。
