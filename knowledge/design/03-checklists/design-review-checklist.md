# Design Review Checklist

## 概述

设计评审是确保产品设计质量的关键环节。本检查清单覆盖视觉一致性、无障碍性、响应式适配、品牌规范、动效表现和交互反馈六大维度，适用于 UI 设计稿评审、前端实现走查和上线前最终确认。每条检查项标注优先级（P0 必须 / P1 应该 / P2 建议），团队可根据项目阶段灵活裁剪。

---

## 一、视觉一致性（Visual Consistency）

### 颜色系统

- [ ] **[P0]** 所有颜色引用 Design Token，无硬编码 HEX/RGB 值
- [ ] **[P0]** 主色/辅色/中性色使用正确的语义色阶（primary-500, neutral-100 等）
- [ ] **[P0]** 语义颜色正确使用：success=绿色、warning=橙色、error=红色、info=蓝色
- [ ] **[P1]** 暗色模式下颜色对比度满足 WCAG AA（4.5:1 正文，3:1 大字）
- [ ] **[P1]** 品牌色在不同背景下可读性测试通过
- [ ] **[P2]** 颜色数量控制在设计系统定义范围内，无多余颜色

### 排版系统

- [ ] **[P0]** 字体族（font-family）与品牌规范一致
- [ ] **[P0]** 标题层级使用正确的 Text Style（H1-H6）
- [ ] **[P0]** 正文字号不小于 14px（移动端不小于 16px）
- [ ] **[P1]** 行高（line-height）遵循设计系统：标题 1.2-1.3，正文 1.5-1.7
- [ ] **[P1]** 字重梯度合理：Regular(400)、Medium(500)、Semibold(600)、Bold(700)
- [ ] **[P1]** 段落最大宽度不超过 75 个字符（中文约 35 个汉字）
- [ ] **[P2]** 中英文混排时字体回退链正确

### 间距与网格

- [ ] **[P0]** 间距使用 4px/8px 基准网格
- [ ] **[P0]** 组件内部间距统一（padding 引用 Token）
- [ ] **[P0]** 组件之间间距统一（margin/gap 引用 Token）
- [ ] **[P1]** 页面整体使用网格系统（12 列或自定义）
- [ ] **[P1]** 内容区域左右边距与设计系统一致
- [ ] **[P2]** 视觉重心平衡，留白合理

### 图标与图片

- [ ] **[P0]** 图标风格统一（线性/填充/双色不混用）
- [ ] **[P0]** 图标尺寸成系统：16px / 20px / 24px / 32px
- [ ] **[P1]** 图片比例一致（卡片内图片统一为 16:9 或 4:3）
- [ ] **[P1]** 占位图/缺省图/加载图设计到位
- [ ] **[P2]** 图片压缩和格式优化（WebP 优先）

## 二、无障碍性（Accessibility）

### 感知性

- [ ] **[P0]** 文本颜色与背景对比度 >= 4.5:1（WCAG AA）
- [ ] **[P0]** 非文本元素（图标、按钮）对比度 >= 3:1
- [ ] **[P0]** 信息传递不仅依赖颜色（辅助图标或文本标识）
- [ ] **[P1]** 图片包含有意义的 alt 文本
- [ ] **[P1]** 表单字段有可见的 label（非仅 placeholder）
- [ ] **[P2]** 视频/音频内容提供字幕或文本替代

### 可操作性

- [ ] **[P0]** 可点击元素最小触控区域 44x44px（移动端）
- [ ] **[P0]** 键盘可达：所有交互元素支持 Tab 导航
- [ ] **[P0]** 焦点状态可见且清晰（focus ring）
- [ ] **[P1]** 焦点顺序与视觉顺序一致
- [ ] **[P1]** 无键盘陷阱（Modal 除外，但 Modal 需支持 Esc 关闭）
- [ ] **[P1]** 拖拽操作提供键盘替代方案
- [ ] **[P2]** 动画可通过 prefers-reduced-motion 关闭

### 可理解性

- [ ] **[P0]** 错误信息明确指出问题和修复方法
- [ ] **[P0]** 表单验证在提交前提供实时反馈
- [ ] **[P1]** 页面语言正确设置（lang 属性）
- [ ] **[P1]** 链接文本描述目标而非泛泛的"点击这里"
- [ ] **[P2]** 复杂操作提供帮助文本或引导

### 健壮性

- [ ] **[P0]** 语义化 HTML：heading 层级正确，表单使用 fieldset/legend
- [ ] **[P0]** ARIA 属性正确使用（aria-label, aria-expanded, role 等）
- [ ] **[P1]** 屏幕阅读器测试通过（VoiceOver / NVDA）
- [ ] **[P2]** 支持浏览器缩放至 200% 仍可用

## 三、响应式适配（Responsive Design）

### 断点系统

- [ ] **[P0]** 定义的断点覆盖主流设备：Mobile / Tablet / Desktop / Wide
- [ ] **[P0]** 断点值与 CSS 框架和设计稿一致
- [ ] **[P1]** 断点之间的过渡平滑，无布局跳变
- [ ] **[P1]** 容器查询用于组件级响应式（如适用）

### 布局适配

- [ ] **[P0]** 移动端优先设计，内容不被截断或溢出
- [ ] **[P0]** 导航在移动端转为汉堡菜单或 Bottom Tab
- [ ] **[P0]** 表格在小屏幕有滚动或卡片化方案
- [ ] **[P1]** 侧边栏在移动端可收起或转为全屏
- [ ] **[P1]** 图片使用 srcset 提供不同分辨率版本
- [ ] **[P2]** 超宽屏（>1920px）有最大内容宽度限制

### 文本适配

- [ ] **[P0]** 长文本正确截断（ellipsis 或 line-clamp）
- [ ] **[P0]** 多语言场景下文本溢出处理
- [ ] **[P1]** 字号在移动端适当缩小（fluid typography）
- [ ] **[P2]** 表单 label 在窄屏时从左侧切换为顶部

## 四、品牌规范（Brand Compliance）

### 视觉标识

- [ ] **[P0]** Logo 使用正确版本（全彩/单色/反白），安全区域充足
- [ ] **[P0]** 品牌主色使用正确，无色差
- [ ] **[P1]** 品牌字体正确加载，回退字体合理
- [ ] **[P1]** 图片风格与品牌调性一致（摄影/插画/3D）
- [ ] **[P2]** 品牌声音（Voice & Tone）在文案中一致

### 风格一致性

- [ ] **[P0]** 按钮/输入框/卡片风格在全站统一
- [ ] **[P0]** 圆角策略统一（全圆角/小圆角/直角不混用）
- [ ] **[P1]** 阴影层级统一（elevation system）
- [ ] **[P1]** 分隔线/边框使用一致
- [ ] **[P2]** 空状态插画风格与品牌一致

### 反 AI 模板感

- [ ] **[P0]** 无默认紫色渐变背景
- [ ] **[P0]** 无 emoji 图标替代专业图标
- [ ] **[P0]** 无默认系统字体堆砌的模板感
- [ ] **[P1]** 有自定义设计元素而非纯 UI Kit 拼搭
- [ ] **[P1]** Hero 区域有真实内容而非泛泛口号
- [ ] **[P2]** 微交互有品牌特色而非通用动效

## 五、动效设计（Motion Design）

### 基础规范

- [ ] **[P0]** 动效时长合理：微交互 100-200ms，过渡 200-400ms，复杂动画 400-600ms
- [ ] **[P0]** 缓动函数统一定义：ease-out 进入、ease-in 退出
- [ ] **[P1]** 页面切换有过渡效果（非瞬切）
- [ ] **[P1]** 列表项入场有交错动画（stagger）
- [ ] **[P2]** 滚动驱动的动画有明确触发点

### 性能约束

- [ ] **[P0]** 动画仅使用 transform 和 opacity（GPU 加速）
- [ ] **[P0]** 无布局抖动（Layout Shift）动画
- [ ] **[P1]** 动画帧率 >= 60fps
- [ ] **[P1]** 首屏加载无阻塞动画
- [ ] **[P2]** Lottie/Rive 动画文件大小 < 50KB

### 有意义的动效

- [ ] **[P0]** 动效传达层级关系（展开/收起、进入/退出）
- [ ] **[P1]** 状态变化有过渡（颜色、尺寸、位置）
- [ ] **[P1]** 加载状态有骨架屏或进度指示
- [ ] **[P2]** 空状态有引导动画

---

## 六、交互反馈（Interaction Feedback）

### 即时反馈

- [ ] **[P0]** 按钮点击有视觉反馈（active state）
- [ ] **[P0]** 表单提交有 Loading 状态和结果反馈
- [ ] **[P0]** 操作成功/失败有 Toast 或 Snackbar 通知
- [ ] **[P1]** 输入框聚焦有边框高亮
- [ ] **[P1]** 链接悬停有下划线或颜色变化

### 防错设计

- [ ] **[P0]** 危险操作（删除、支付）有二次确认
- [ ] **[P0]** 表单有实时校验而非仅提交后校验
- [ ] **[P1]** 不可逆操作有撤销机制
- [ ] **[P1]** 离开未保存页面有确认提示
- [ ] **[P2]** 自动保存草稿

### 系统状态

- [ ] **[P0]** 加载中有 Loading 指示器（Spinner / Skeleton）
- [ ] **[P0]** 空数据有 Empty State 说明和引导操作
- [ ] **[P0]** 网络错误有友好提示和重试按钮
- [ ] **[P1]** 长列表有分页或无限滚动指示
- [ ] **[P1]** 后台任务有进度条或完成通知
- [ ] **[P2]** 实时数据有更新指示器

## 评审流程

### 评审时机

1. **设计评审**：设计稿完成后、开发前
2. **实现走查**：前端实现后、合并前
3. **上线确认**：Staging 环境、正式上线前

### 评审参与者

| 角色 | 职责 |
|------|------|
| 设计师 | 主导评审，检查还原度 |
| 前端开发 | 说明技术约束，确认可行性 |
| PM | 确认功能完整性和业务逻辑 |
| QA | 补充边界用例和兼容性 |

### 问题分级处理

- **P0 Blocker**：必须在当前迭代修复，阻止上线
- **P1 Major**：当前迭代修复，不阻止灰度但阻止全量
- **P2 Minor**：排入下个迭代的优化 Backlog

## Agent Checklist

以下为 AI Agent 在执行设计评审时的检查要点：

- [ ] 使用 Design Token 验证颜色/字体/间距一致性
- [ ] 通过对比度工具检查无障碍合规性（axe / Lighthouse）
- [ ] 在 Mobile / Tablet / Desktop 三个视口验证响应式
- [ ] 确认品牌元素（Logo/主色/字体）使用正确
- [ ] 检查动效性能（仅 transform/opacity，无 Layout Shift）
- [ ] 验证所有交互元素有完整反馈状态
- [ ] 确认无 AI 模板感（无紫色渐变/emoji 图标/默认字体）
- [ ] P0 问题全部修复后方可通过评审
- [ ] 评审结果记录并归档至项目文档
