# 鸿蒙设计规范（HarmonyOS Design · 官方）

> 鸿蒙 App 不能套安卓/iOS 的视觉与交互。HarmonyOS Design 是华为官方设计语言，遵循它才像"纯血鸿蒙"。设计鸿蒙应用前必读。

## 1. 设计语言与原则

- HarmonyOS Design 强调**视觉连续、界面关联、一致体验**；系统色调受"鸿蒙宇宙"世界观启发（宇宙蓝、雪域灰为基础）。
- 规范的按钮位置、菜单布局、操作手势，降低用户学习成本；交互符合鸿蒙用户习惯。
- 简洁、聚焦内容、层次清晰。

## 2. 字体与排版

- 用官方 **鸿蒙 Sans (HarmonyOS Sans)** 字体（华为开发者联盟提供下载）；清晰字阶层级。
- 字号/行高按规范，支持系统字体大小调节（无障碍）。

## 3. 色彩

- 用**系统色彩体系 + 品牌色**，注意品牌色与系统色对比，保证视觉连续与可读性。
- 支持深浅色模式；语义化用色而非硬编码。

## 4. 栅格与自适应布局（鸿蒙强项：一次开发多端）

- 用**自适应栅格布局**（栅格系统 + 比例缩放 + 断点）适配手机/折叠屏/平板/PC/车机等多设备形态。
- **一次开发多端部署**：同一套设计随设备形态自适应（响应式断点、`mediaquery`），针对**折叠屏**做展开/折叠态适配。
- 不写死像素；用逻辑栅格与弹性布局。

## 5. 组件与资源（用官方资源）

- 用 **ArkUI 系统组件**与官方组件规范，自带正确外观/交互/无障碍。
- 用官方**图标库**（HarmonyOS 在线图标资源）与**音效库**，不要用 emoji 当功能图标。
- 导航/Tab/列表/弹窗等遵循鸿蒙交互范式。

## 6. 交互与动效

- 操作手势、转场动效遵循 HarmonyOS Design（自然、有层次、表达关联）。
- 即时反馈、加载/空/错误三态、触感反馈。

## 7. 反模式（出现即不合格 / "不像鸿蒙"）

- 套用安卓 Material 或 iOS HIG 的视觉/交互范式而非 HarmonyOS Design。
- 不用鸿蒙 Sans/官方图标；emoji 当图标。
- 不做多设备/折叠屏自适应（写死手机尺寸）；硬编码颜色不适配深色。
- 不支持系统字号调节；忽略无障碍。

## 8. 最低交付 checklist

- [ ] 遵循 HarmonyOS Design 视觉/交互语言（宇宙蓝/雪域灰基调、视觉连续）。
- [ ] 鸿蒙 Sans 字体 + 清晰字阶 + 支持系统字号；语义色彩 + 深色适配。
- [ ] 自适应栅格 + 一次开发多端 + 折叠屏展开/折叠态适配，不写死像素。
- [ ] ArkUI 系统组件 + 官方图标库(无 emoji) + 鸿蒙交互范式。
- [ ] 自然动效 + 三态 + 触感反馈 + 无障碍。
