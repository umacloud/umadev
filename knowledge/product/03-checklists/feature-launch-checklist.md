# Feature Launch Checklist

## 概述

功能上线是产品交付中风险最集中的环节。本检查清单覆盖从需求确认到全量发布的完整链路，分为七个阶段：需求确认、设计评审、开发完成、测试通过、灰度发布、监控告警和文档更新。每条检查项标注责任人角色和优先级（P0 必须 / P1 应该 / P2 建议），确保关键环节不被遗漏。

---

## 一、需求确认（Requirement Confirmation）

### 需求文档

- [ ] **[P0][PM]** PRD 文档已发布并经相关方评审
- [ ] **[P0][PM]** 用户故事（User Story）有明确的验收标准（Acceptance Criteria）
- [ ] **[P0][PM]** 核心功能的边界情况已列举（空数据、超长输入、并发操作等）
- [ ] **[P1][PM]** 功能范围（Scope）明确，有"不做"清单
- [ ] **[P1][PM]** 依赖项已识别（后端 API、第三方服务、数据迁移等）
- [ ] **[P2][PM]** 竞品分析已完成，差异化定位清晰

### 优先级与排期

- [ ] **[P0][PM]** 功能优先级确认（P0/P1/P2），与团队达成共识
- [ ] **[P0][TL]** 技术方案已评审，工作量估算完成
- [ ] **[P1][PM]** 上线日期确认，倒推里程碑节点
- [ ] **[P1][PM]** 与运营/市场同步上线计划
- [ ] **[P2][PM]** 风险评估完成，有应急预案

### 合规与安全

- [ ] **[P0][Security]** 涉及用户数据的功能通过安全评审
- [ ] **[P0][Legal]** 涉及隐私协议变更的内容法务确认
- [ ] **[P1][PM]** 合规要求梳理（GDPR / 等保 / 行业规范）
- [ ] **[P2][PM]** 数据埋点方案确认，不采集无关个人信息

## 二、设计评审（Design Review）

### 视觉设计

- [ ] **[P0][Design]** 设计稿覆盖所有页面和关键状态
- [ ] **[P0][Design]** 组件使用设计系统标准组件，无自造组件
- [ ] **[P0][Design]** 颜色/字体/间距引用 Design Token
- [ ] **[P1][Design]** 响应式适配：Mobile + Desktop 两套设计稿
- [ ] **[P1][Design]** 空状态、错误状态、加载状态设计完成
- [ ] **[P2][Design]** 动效交互标注（时长/缓动/触发条件）

### 交互设计

- [ ] **[P0][Design]** 核心用户流程已绘制并走查
- [ ] **[P0][Design]** 所有交互元素标注状态（hover/focus/disabled）
- [ ] **[P1][Design]** 边界情况交互定义（网络异常、超时、重复操作）
- [ ] **[P1][Design]** 无障碍设计考量（对比度、触控区域、键盘导航）
- [ ] **[P2][Design]** 用户引导/Onboarding 方案

### 设计交付

- [ ] **[P0][Design]** 设计稿已标注并分享给开发团队
- [ ] **[P0][Design]** Design Token 已导出并同步
- [ ] **[P1][Design]** 组件映射表已建立（Figma 组件 <-> 前端组件）
- [ ] **[P1][Design]** 设计交付会议已完成

## 三、开发完成（Development Complete）

### 代码质量

- [ ] **[P0][Dev]** 代码通过 Lint 检查（ESLint / Ruff / 团队规则）
- [ ] **[P0][Dev]** 代码通过类型检查（TypeScript / mypy）
- [ ] **[P0][Dev]** 无硬编码敏感信息（密钥、密码、Token）
- [ ] **[P0][Dev]** 代码评审（Code Review）已完成且通过
- [ ] **[P1][Dev]** 函数/方法有合理的注释和文档字符串
- [ ] **[P1][Dev]** 复杂逻辑有内联注释说明
- [ ] **[P2][Dev]** 技术债务已记录到 Backlog

### 功能实现

- [ ] **[P0][Dev]** 所有 Acceptance Criteria 实现完毕
- [ ] **[P0][Dev]** API 契约与前后端约定一致
- [ ] **[P0][Dev]** 数据库变更（Migration）已准备且可回滚
- [ ] **[P1][Dev]** 功能开关（Feature Flag）已配置，支持灰度
- [ ] **[P1][Dev]** 国际化（i18n）文案已提取
- [ ] **[P2][Dev]** 性能优化已完成（懒加载/缓存/查询优化）

### 前端实现

- [ ] **[P0][FE]** 设计走查第一轮完成（布局/颜色/字体）
- [ ] **[P0][FE]** 响应式在 Mobile / Desktop 上验证通过
- [ ] **[P0][FE]** 交互状态完整（loading/empty/error/success）
- [ ] **[P1][FE]** 浏览器兼容性测试（Chrome/Firefox/Safari/Edge）
- [ ] **[P1][FE]** 首屏加载时间 < 3 秒（4G 网络）
- [ ] **[P2][FE]** Lighthouse Performance Score >= 80

### 后端实现

- [ ] **[P0][BE]** API 响应时间满足 SLA（P95 < 500ms）
- [ ] **[P0][BE]** 接口幂等性处理（防重复提交）
- [ ] **[P0][BE]** 输入验证和防注入处理
- [ ] **[P1][BE]** 限流策略配置（Rate Limiting）
- [ ] **[P1][BE]** 日志格式规范，包含请求追踪 ID
- [ ] **[P2][BE]** 慢查询优化（SQL Explain Plan 审查）

## 四、测试通过（Testing Passed）

### 单元测试

- [ ] **[P0][Dev]** 所有单元测试通过
- [ ] **[P1][Dev]** 边界情况有测试用例
- [ ] **[P2][Dev]** 新增代码增量覆盖率 >= 90%

### 集成测试

- [ ] **[P0][QA]** 核心用户流程端到端测试通过
- [ ] **[P0][QA]** API 接口测试通过（Postman / 自动化）
- [ ] **[P1][QA]** 前后端联调测试通过
- [ ] **[P1][QA]** 第三方依赖集成测试通过

### 功能测试

- [ ] **[P0][QA]** 所有 Acceptance Criteria 验证通过
- [ ] **[P0][QA]** 边界情况测试通过（空值/最大值/特殊字符）
- [ ] **[P0][QA]** 权限控制验证（不同角色访问控制）
- [ ] **[P1][QA]** 多浏览器/多设备兼容性测试
- [ ] **[P1][QA]** 数据迁移/升级路径测试
- [ ] **[P2][QA]** 探索性测试完成

### 非功能测试

- [ ] **[P0][QA]** 安全测试：XSS / CSRF / SQL 注入扫描通过
- [ ] **[P1][QA]** 性能测试：并发 100 用户响应时间满足 SLA
- [ ] **[P1][QA]** 压力测试：识别系统瓶颈和崩溃阈值
- [ ] **[P2][QA]** 无障碍测试：axe / Lighthouse Accessibility >= 90

### 回归测试

- [ ] **[P0][QA]** 自动化回归测试套件全部通过
- [ ] **[P0][QA]** 关联功能手动回归测试通过
- [ ] **[P1][QA]** 无新增 P0/P1 Bug

## 五、灰度发布（Gradual Rollout）

### 发布准备

- [ ] **[P0][DevOps]** 部署脚本/Pipeline 已测试
- [ ] **[P0][DevOps]** 回滚方案已准备并验证
- [ ] **[P0][Dev]** 数据库 Migration 已在 Staging 验证
- [ ] **[P1][DevOps]** 发布窗口确认（避开高峰期）
- [ ] **[P1][PM]** 发布通知已发送给相关团队

### 灰度策略

- [ ] **[P0][PM]** 灰度策略确定（按用户比例/按地区/按租户）
- [ ] **[P0][Dev]** Feature Flag 已配置，支持动态开关
- [ ] **[P0][PM]** 灰度范围：1% -> 5% -> 20% -> 50% -> 100%
- [ ] **[P1][PM]** 每个灰度阶段的观察期和通过标准定义
- [ ] **[P1][QA]** 灰度期间的手动抽查计划

### 灰度验证

- [ ] **[P0][Dev]** 灰度 1% 上线后核心指标正常（错误率/延迟/CPU）
- [ ] **[P0][QA]** 灰度用户功能验证通过
- [ ] **[P0][PM]** 无用户投诉或负面反馈
- [ ] **[P1][Dev]** 日志中无异常报错
- [ ] **[P1][PM]** 业务指标（转化率/使用率）符合预期

## 上线后跟踪（Post-Launch）

### 第一天（D+1）

- [ ] 核心指标无异常（错误率 < 0.1%，延迟 P95 < 500ms）
- [ ] 无 P0 用户反馈
- [ ] 灰度比例按计划推进

### 第一周（D+7）

- [ ] 灰度达到 50% 或全量
- [ ] 业务指标达到预期基线
- [ ] 已知问题清单记录并排期
- [ ] 上线复盘会议完成

### 第一月（D+30）

- [ ] 全量发布完成
- [ ] 用户反馈收集并分析
- [ ] 功能使用数据分析（是否达到目标）
- [ ] 技术债务清理完成
- [ ] 下一迭代优化方向确定

## Agent Checklist

以下为 AI Agent 在辅助功能上线时的检查要点：

- [ ] 确认 PRD 和 Acceptance Criteria 已明确
- [ ] 确认设计稿覆盖所有状态并通过评审
- [ ] 确认代码通过 Lint、类型检查和 Code Review
- [ ] 确认核心逻辑单元测试覆盖率 >= 80%
- [ ] 确认端到端测试和安全扫描通过
- [ ] 确认 Feature Flag 已配置，支持灰度和回滚
- [ ] 确认监控告警已配置（错误率/延迟/资源使用）
- [ ] 确认回滚方案已准备并验证
- [ ] 确认用户文档和 API 文档已更新
- [ ] 确认上线后跟踪计划已安排（D+1/D+7/D+30）
