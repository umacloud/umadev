# KPI 与指标树方法库（商业级必读）

> 没有指标树的产品，"做得好不好"只能靠感觉和嗓门吵出来。最常见的失败有两种：一是只盯**虚荣指标**（PV/UV/下载量），看着涨却和价值无关；二是各团队**口径打架**，同一个"活跃用户"三套算法，开会先吵定义。指标树的作用是把一个模糊的战略目标，**逐层拆成可观测、可归因、有人负责的具体指标**，让每个功能上线后都能回答"它到底有没有用、用在哪"。本方法库给出指标分层、构建步骤、口径治理与验收门禁。

## 1. 指标分层（从战略到行为，逐层可拆）

| 层级 | 作用 | 示例 |
|---|---|---|
| **北极星指标** | 全公司/产品线唯一对齐的核心价值度量 | 周活跃创作数、成功交易额、完成核心任务的用户数 |
| **一级指标（输入）** | 直接驱动北极星的可拆解因子 | 新增、激活、留存、收入、成本 |
| **二级指标（健康度）** | 反映各环节质量与效率 | 转化率、漏斗流失率、任务成功率、P95 时延、错误率 |
| **过程指标（行为）** | 最细粒度的可观测行为，用于定位 | 页面点击、操作耗时、失败类型分布、空结果率 |

**北极星指标的判定**：它涨，用户拿到的价值就真的涨（不是仅仅"用得更久")；它能被一级指标解释；它不能靠刷量伪造。下载量、PV 几乎都不合格。

## 2. 指标树构建步骤

1. **锚定北极星 + 时间窗**：先定义"我们为用户创造的核心价值"，再选一个能代表它的可度量量，明确统计窗口（日/周/月）。
2. **按驱动关系拆一级指标**：把北极星写成乘法/加法分解（如 收入 = 活跃用户 × 付费转化率 × 客单价），每个因子就是一级指标。
3. **沿关键用户路径拆二级/过程指标**：对每条核心任务流（注册→激活→留存→付费），在每个节点定义成功指标与失败指标。
4. **为每个叶子节点定义口径与阈值**：口径说明、数据源、更新频率、负责人、健康/告警阈值。
5. **反向校验**：每个一级/二级指标都要能回答"它服务于哪个上层指标"，挂不上北极星的指标要么删掉，要么补一层。

## 3. 成功指标 vs 失败指标（必须成对定义）

只统计"做成了多少"会掩盖问题，每个关键环节都要同时盯失败面：

| 环节 | 成功指标 | 失败指标 |
|---|---|---|
| 搜索 | 搜索后点击率 | 空结果率、零点击率 |
| 表单/结算 | 提交成功率 | 字段报错率、放弃率、超时率 |
| 任务流 | 端到端完成率 | 中途流失节点分布、平均重试次数 |
| 服务质量 | P50/P95 时延达标率 | 错误率、超时率、降级触发次数 |

## 4. 口径治理（指标可信的前提）

每个指标必须登记为一条可治理的元数据，否则不予上线：

- **口径定义**：精确到"谁、在什么时间窗、满足什么条件算一次"——"活跃"是打开 App 还是完成一次核心动作，必须写死。
- **数据源 + 计算逻辑**：来源表/事件、过滤条件、去重规则、时区，全部明确，可复算。
- **更新频率 + 负责人**：实时/小时/天级，谁对这个数负责、谁有权改口径。
- **跨团队唯一口径**：同一指标名全公司只能有一套定义；要拆分必须改名（如"登录活跃"vs"创作活跃"），禁止同名异义。
- **历史基线对齐**：上生产前用历史数据跑一遍，确认数值落在合理区间、与已有口径不冲突。

## 5. 指标验收门禁（功能上线前）

- 每个新功能上线前**至少绑定 1 个业务指标 + 2 个质量指标**，并预设预期变化方向与幅度。
- 指标缺口径说明 / 数据源 / 负责人任意一项 → 不允许进生产。
- 关键指标必须配**告警阈值**与异常下钻路径，能从二级指标一路下钻到过程指标定位根因。
- 上线后设**验证窗口**：到期用绑定指标判定功能成败，达不到预期触发迭代或回滚，而不是"上了就不管"。

## 反模式

- **虚荣指标驱动**：只看 PV/UV/下载量等看着好看、与价值无关的数，掩盖真实的任务失败。
- **只统计成功面**：盯着提交成功率却不看放弃率/报错率，问题被平均数藏住。
- **口径跨团队打架**：同名指标多套算法，开会先吵定义，决策建立在不可比的数上。
- **指标挂不上北极星**：堆了一屏指标，没人能说清它服务于哪个上层目标，沦为数据噪声。
- **无负责人无更新频率**：指标没人维护，口径悄悄漂移，数据慢慢失真却无人察觉。
- **上线即弃**：功能上线绑了指标却不设验证窗口，从不回看到底有没有效果。

## 最低交付 checklist

- [ ] 产品/产品线有唯一北极星指标，且能被一级指标解释、不可刷量伪造。
- [ ] 指标树自上而下分层（北极星 → 一级 → 二级 → 过程），每个下层指标可追溯到上层。
- [ ] 关键环节均成对定义成功指标与失败指标。
- [ ] 每个指标登记了口径、数据源、计算逻辑、更新频率、负责人，且跨团队口径唯一。
- [ ] 所有指标上生产前完成历史基线对齐。
- [ ] 每个新功能绑定 ≥1 业务指标 + ≥2 质量指标，并预设预期方向与验证窗口。
- [ ] 关键指标配置告警阈值与从二级到过程指标的下钻定位路径。
