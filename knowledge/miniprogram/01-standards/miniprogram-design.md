# 微信小程序设计规范（官方设计指南 + WeUI）

> 微信官方有明确的小程序设计指南。遵循它做出**友好、高效、一致**、符合微信生态习惯的体验；用 WeUI 对齐官方视觉。

## 1. 核心理念（官方设计指南）

- 建立在**充分尊重用户知情权与操作权**的基础上：清楚告知在做什么、要什么权限、用户可控可退。
- 在微信生态内追求**友好、高效、一致**的体验——符合用户在微信里的既有习惯，降低学习成本。
- **即用即走**：聚焦核心任务，轻量、快、不强留；首屏直达价值。

## 2. 视觉与组件（用 WeUI）

- 用微信官方 **WeUI**（官方设计团队为小程序量身设计的组件库 / weui-wxss + WeUI 扩展组件 / Vant Weapp 等）保证视觉与微信原生一致。
- 遵循微信视觉规范：克制配色、标准间距、清晰层级；主色不滥用；不要做得"很不微信"。
- 图标用图标库（不用 emoji 当功能图标）；尺寸用 **rpx** 适配不同屏宽。

## 3. 导航与布局

- 顶层用 **TabBar**（底部，2–5 个）；页面内层级用导航栏返回。
- 遵循微信导航与手势习惯（右滑返回）；不要自造反直觉导航。
- 关键操作位置符合习惯；表单/列表/卡片遵循 WeUI 规范。

## 4. 交互与体验

- 每个操作有反馈（loading / toast / 按钮态）；**加载/空/错误三态**都要做。
- 首屏骨架屏/loading；下拉刷新、触底加载；秒开（缓存 + 数据预取）。
- 授权时机合理：在用户理解为什么需要时才申请（如点"获取位置"才请求），尊重拒绝并降级。

## 5. 合规（设计层面）

- 隐私协议、用户授权弹窗符合微信与法规要求；明确数据用途。
- 遵守微信**审核与运营规范**（类目资质、内容、诱导分享限制等），避免设计出违规交互。

## 6. 反模式（出现即不合格 / "不像微信"）

- 不用 WeUI/微信视觉，做成 web 或他端风格，体验割裂。
- 自造反直觉导航/手势；TabBar 超 5 个或层级混乱。
- 不做三态/反馈/骨架屏；不尊重知情权，进来就索权/强留。
- emoji 当图标；不用 rpx 适配导致小屏错位。

## 7. 最低交付 checklist

- [ ] 尊重知情权/操作权；友好高效一致、即用即走、首屏直达。
- [ ] 用 WeUI/微信视觉规范；克制配色 + 清晰层级 + rpx 适配；图标库无 emoji。
- [ ] TabBar(≤5)+导航返回符合微信习惯。
- [ ] 三态 + 反馈 + 骨架屏 + 秒开；授权合理时机 + 降级。
- [ ] 隐私协议/授权合规 + 遵守审核运营规范。
